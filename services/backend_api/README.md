# ROM Tracker Backend API

This backend is a lightweight MVP implementation for the ROM Tracker monorepo.

It focuses on the critical project flow:

1. Login
2. Load current user
3. Load doctors
4. Load doctor details and slots
5. Create booking
6. Load patient sessions
7. Load doctor sessions
8. Update session status
9. Submit AI exercise result and receive a generated report

## Stack

- Python 3.9+
- FastAPI
- SQLite

## Local Run

```powershell
cd D:\rom_tracker_app\services\backend_api
py -m pip install -r requirements.txt
py -m uvicorn app.main:app --reload --host 0.0.0.0 --port 3000
```

## AI Result Flow

The backend is ready now even if the Unity AI module has not been delivered yet.

Expected payload from Unity or any temporary mock sender:

```json
{
  "exercise": "Squat",
  "reps": 12,
  "timestamp": "2026-04-25T10:30:00Z"
}
```

The backend will:

1. attach the result to the session
2. compare it against the patient's real previously recorded results for the same exercise
3. generate a report using real recorded history such as:
   - total recorded sessions
   - total recorded reps
   - average reps
   - best reps
   - previous reps
   - trend
4. return the saved result plus the report

## Product Logic Already Implemented

- Booking creates notifications for both patient and doctor.
- Session start creates a doctor notification.
- Completing or canceling a session creates notifications for both sides.
- Reviews are limited to completed sessions and create a doctor notification.
- Wishlist is stored per patient and linked to real doctor records.
- AI reports are based on the patient's actual recorded history for the same exercise, not only the latest single result.

## Notes

- Tokens are demo tokens generated by the backend and kept in memory for the running process.
- SQLite data is stored locally in `services/backend_api/data/`.
- This is enough for development and demo purposes without any paid hosting.
